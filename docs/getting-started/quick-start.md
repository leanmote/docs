# Quick Start

This guide takes your organization from first login to an actionable dashboard baseline.

## 1. Access and organization setup

1. Sign in at `https://app.leanmote.com`.
2. Create a new organization or accept your workspace invite.
3. Confirm admin ownership and default organization profile settings.

## 2. Connect productivity tools

In **Settings > Productivity Tools**, connect at least:

- One source control provider (GitHub, GitLab, or Azure DevOps)
- One planning provider (Jira, Linear, ClickUp, or Azure Boards)

Then select only the repositories/projects you want to track in Leanmote.

## 3. Confirm security and access model

Before enabling full sync:

- Review provider permission scopes.
- Confirm least-privilege service account access.
- Validate who can manage integrations and settings.

For full details, see [Administration: Integrations](../administration/integrations.md).

## 4. Validate data readiness

Ensure:

- Team and user mappings are correct.
- Status mapping is configured for board workflows.
- Initial sync completed successfully.
- Expected backfill window is available for your plan.

## 5. Open your first dashboards

Start with:

- [Performance Delivery](../dashboards/performance-delivery.md)
- [Project Financials](../dashboards/project-financials.md)

Apply filters for date range, teams, and repositories/projects to establish your baseline.

## 6. Run a weekly operating rhythm

- Weekly: delivery and bottleneck review by team
- Bi-weekly: mapping and workflow quality checks
- Monthly: strategic KPI and investment review
