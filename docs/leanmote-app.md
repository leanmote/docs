# Using app.leanmote.com

This guide describes the main areas of the Leanmote web application and how teams use them in day-to-day operations.

## Main navigation areas

| Area | Purpose |
|------|---------|
| Dashboards | Track DORA, flow, and team execution metrics |
| Integrations | Connect and manage data sources |
| Teams | Organize members and ownership boundaries |
| Settings | Configure access, tokens, and organization-level preferences |

## Dashboards

Dashboards are where most teams run delivery reviews.

Common usage patterns:

- Weekly engineering review by team
- Monthly leadership trend review
- Incident recovery and delivery-risk follow-up

Recommended dashboard views:

- Delivery health (DORA + flow)
- Team execution view (throughput, cycle time, WIP)
- Collaboration view (review participation and interaction patterns)

## Integrations

Leanmote supports cross-system analytics by syncing data from source control, planning, and collaboration tools.

Typical integration sequence:

1. Connect source control
2. Connect planning system
3. Connect communication tools
4. Validate team and repository/project mappings

## Team and access management

Keep your workspace accurate over time:

- Maintain team ownership and membership
- Review role permissions after org changes
- Archive teams no longer in active delivery scope

## Operating model recommendation

Use Leanmote as part of a recurring cadence:

- Weekly: team-level metrics review
- Bi-weekly: bottleneck and workflow validation
- Monthly: leadership KPI review and target adjustment

## Outcomes you should expect

- Faster identification of delivery bottlenecks
- Better alignment between managers and executives
- More consistent prioritization based on measurable trends
