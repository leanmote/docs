# Settings

The Settings area controls how Leanmote behaves for your entire organization.

## Organization controls

Use settings to standardize:

- Timezone and reporting windows
- Default dashboard and filter behavior
- API token governance
- Integration management permissions

## Metric visibility and custom filters

You can define custom filters to hide or segment metrics across the company.

Common reasons:

- Exclude deprecated repositories or teams.
- Separate experimentation streams from core delivery.
- Limit visibility for sensitive operational slices.

When using organization-wide filters, document the policy so all teams interpret metrics consistently.

## Recommended settings governance

- Review defaults monthly.
- Keep token and access ownership explicit.
- Document any global filter that changes KPI interpretation.
- Re-validate settings after major org or workflow changes.
